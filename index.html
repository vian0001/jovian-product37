1. KONSEP DASAR ROUTING IP

Pengertian Routing IP

Dari sumber MikroTik Documentation (2024), routing adalah proses pemilihan jalur untuk meneruskan paket data antar jaringan yang berbeda. Router MikroTik menjalankan proses ini dengan menyimpan semua informasi routing dalam database yang disebut RIB (Routing Information Base), yang berisi daftar entri untuk tujuan jaringan tertentu dan gateway mereka (alamat perangkat berikutnya dalam jalur atau next-hop).

Dari sumber Tech Layer-X (2025), routing pada dasarnya merupakan proses pemilihan jalur untuk meneruskan paket data antar jaringan yang berbeda. MikroTik RouterOS mendukung baik static routing—di mana rute didefinisikan secara manual—maupun dynamic routing—di mana rute dipelajari dan diperbarui secara otomatis menggunakan protokol routing.

Komponen Utama Routing pada MikroTik

Dari sumber MikroTik Documentation (2024), MikroTik RouterOS menggunakan beberapa komponen utama dalam proses routing:

RIB (Routing Information Base) - Berisi semua prefix yang dipelajari dari protokol routing
# Mencakup rute dari jaringan terkoneksi (connected), statis, BGP, RIP, OSPF
FIB (Forwarding Information Base) - Tabel yang digunakan router untuk meneruskan paket
# Berisi rute aktif yang dipilih dari RIB
Route - Entri individual dalam tabel routing yang menunjukkan jalur ke jaringan tujuan
# Berisi dst-address, gateway, distance, dan properti lainnya

Dari sumber Cyberlab Pacific (2025), MikroTik menggunakan CLI (Command Line Interface) yang kuat untuk mengonfigurasi routing, dengan perintah-perintah yang diatur dalam hierarki menu. Semua informasi routing dapat diakses melalui perintah dasar /ip route yang memungkinkan administrator untuk mengelola entri routing.

Kesimpulan: Dari 2 pendapat tersebut dapat disimpulkan bahwa routing pada MikroTik adalah proses pemilihan jalur untuk meneruskan paket data antar jaringan, yang dilakukan dengan bantuan database routing (RIB dan FIB). MikroTik RouterOS menawarkan fleksibilitas dengan mendukung baik static routing maupun dynamic routing, dan menyediakan interface CLI yang kuat untuk konfigurasi dan pengelolaan routing.

2. STATIC ROUTING

Tampilan topologi di gns3

Pengertian Static Routing

Dari sumber MikroTik Wiki (2024), Static Routing adalah metode routing di mana rute ditentukan secara manual oleh administrator jaringan. Rute statis menentukan jalur yang akan digunakan paket data untuk mencapai jaringan tujuan tertentu. Pada MikroTik, rute statis memiliki prioritas lebih tinggi daripada rute dinamis karena memiliki nilai distance yang lebih rendah secara default.

Dari sumber Tech Layer-X (2025), Static Routing paling cocok untuk jaringan kecil atau stabil di mana jalur jarang berubah. Metode ini memberikan kontrol penuh atas keputusan routing, tetapi memerlukan pembaruan manual ketika topologi jaringan berubah.

Konfigurasi Static Route di MikroTik

Dari sumber MikroTik Documentation (2024), untuk menambahkan rute statis pada MikroTik, kita menggunakan perintah /ip route add dengan parameter dst-address dan gateway:

/ip route add dst-address=192.168.2.0/24 gateway=172.16.1.2
# Menambahkan rute statis untuk jaringan 192.168.2.0/24 melalui gateway 172.16.1.2
/ip route print
# Perintah untuk melihat tabel routing
Flags: D - dynamic; X - disabled, I - inactive, A - active; C - connect, S - static, r - rip, b - bgp, o - ospf, d - dhcp, v - vpn
# DST-ADDRESS GATEWAY Distance
0 DAC 10.1.1.0/24 ether1 0
1 DAC 172.16.1.0/30 ether2 0
2 DAC 192.168.1.0/24 bridge1 0
3 AS 192.168.2.0/24 172.16.1.2 1

Dari sumber RouterX (2025), untuk konfigurasi static route melalui Winbox GUI:

  1. Buka IP > Routes
  2. Klik tombol + untuk menambahkan rute baru
  3. Masukkan alamat tujuan (Dst. Address) dan netmask
  4. Tentukan gateway (alamat IP router berikutnya)
  5. Atur distance jika diperlukan (default: 1)
  6. Klik Apply dan OK

Flag pada output /ip route print memberikan informasi penting tentang status rute:

  • A - Active: Rute aktif dan digunakan
  • S - Static: Rute yang dikonfigurasi secara manual
  • D - Dynamic: Rute yang dipelajari dari protokol routing dinamis
  • C - Connected: Rute yang terhubung langsung ke interface router
  • X - Disabled: Rute yang dinonaktifkan
  • I - Inactive: Rute tidak aktif karena berbagai alasan

Default Route

Dari sumber MikroTik Documentation (2024), Default Route adalah rute khusus dengan alamat tujuan 0.0.0.0/0. Rute ini digunakan sebagai "jalur cadangan" atau "rute terakhir" ketika tidak ada rute spesifik lain yang cocok untuk paket data. Pada MikroTik, default route dapat ditambahkan dengan:

/ip route add dst-address=0.0.0.0/0 gateway=10.0.0.1
# Menambahkan default route menuju gateway 10.0.0.1
/ip route add gateway=10.0.0.1
# Alternatif - dst-address=0.0.0.0/0 dianggap default jika tidak diisi

Dari sumber MikroTik Wiki (2024), Default Route adalah jalur yang akan digunakan untuk semua traffic yang tidak memiliki rute spesifik dalam tabel routing. Default route dikenal juga sebagai "gateway of last resort" dan sangat penting untuk konektivitas internet pada sebagian besar jaringan.

Kesimpulan: Dari 2 pendapat tersebut dapat disimpulkan bahwa Static Routing pada MikroTik adalah metode konfigurasi manual jalur yang akan digunakan untuk mengirimkan paket ke jaringan tujuan. Konfigurasi dapat dilakukan melalui CLI dengan perintah /ip route add atau melalui Winbox GUI. Default Route (0.0.0.0/0) merupakan komponen penting yang menentukan gateway untuk semua traffic yang tidak memiliki rute spesifik, biasanya digunakan untuk koneksi internet.

3. DYNAMIC ROUTING

Tampilan topologi di gns3

Pengertian Dynamic Routing

Dari sumber Tech Layer-X (2025), Dynamic Routing adalah metode routing di mana router secara otomatis belajar, memilih, dan memperbarui rute melalui protokol routing. Protokol ini memungkinkan router untuk berbagi informasi routing dengan router lain dan beradaptasi dengan perubahan topologi jaringan. MikroTik RouterOS mendukung beberapa protokol routing dinamis seperti RIP, OSPF, dan BGP.

Dari sumber MikroTik Documentation (2024), pada Dynamic Routing, router secara otomatis bertukar informasi tentang jaringan yang mereka ketahui, dan berdasarkan informasi ini, mereka membangun dan memperbarui tabel routing mereka. Metode ini sangat berguna untuk jaringan yang besar dan kompleks di mana konfigurasi dan pemeliharaan rute statis akan sangat memakan waktu.

Protokol Dynamic Routing pada MikroTik

Dari sumber Tech Layer-X (2025), MikroTik mendukung beberapa protokol routing dinamis:

RIP (Routing Information Protocol)
# Protokol distance-vector yang menggunakan jumlah hop sebagai metrik
# Cocok untuk jaringan kecil dengan maksimal 15 hop
OSPF (Open Shortest Path First)
# Protokol link-state yang menggunakan algoritma Shortest Path First
# Mendukung desain hierarkis dengan membagi jaringan menjadi area
BGP (Border Gateway Protocol)
# Protokol path-vector untuk routing antar autonomous system (AS)
# Penting untuk routing skala internet dan organisasi dengan multiple IP publik

Dari sumber MikroTik Wiki (2024), OSPF adalah protokol link-state yang mengurus rute dalam struktur jaringan dinamis yang dapat menggunakan berbagai jalur ke subjaringannya. Ini selalu memilih jalur terpendek ke subjaringan terlebih dahulu, dan setiap area menjalankan salinan terpisah dari algoritma routing link-state dasar.

Konfigurasi OSPF pada MikroTik

Dari sumber MikroTik Wiki (2024), berikut adalah langkah-langkah dasar untuk mengonfigurasi OSPF:

/routing ospf instance add name=default router-id=1.1.1.1
# Membuat instance OSPF dengan router-id 1.1.1.1
/routing ospf area add name=backbone area-id=0.0.0.0 instance=default
# Menambahkan area backbone (area 0)
/routing ospf interface-template add area=backbone networks=192.168.1.0/24
# Menambahkan jaringan 192.168.1.0/24 ke area backbone
/routing ospf interface-template add area=backbone networks=192.168.2.0/24
# Menambahkan jaringan 192.168.2.0/24 ke area backbone

Dari sumber Tech Layer-X (2025), untuk memeriksa status OSPF:

/routing ospf neighbor print
# Memeriksa tetangga OSPF
/routing ospf interface print
# Melihat interface OSPF aktif
/routing ospf route print
# Melihat rute yang dipelajari dari OSPF
/routing ospf lsa print
# Melihat database LSA (Link State Advertisement)

OSPF menggunakan konsep area untuk menskalakan jaringan besar. Area 0 atau backbone area harus menjadi pusat dari semua area lain. Untuk koneksi antar area, digunakan router yang disebut Area Border Router (ABR) yang memiliki interface di dua atau lebih area.

Konfigurasi BGP pada MikroTik

Dari sumber FastNetMon (2025), berikut adalah langkah-langkah dasar untuk mengonfigurasi BGP:

/routing bgp instance set default as=65001 router-id=10.0.0.1 redistribute-static=no
# Mengkonfigurasi instance BGP dengan nomor AS 65001
/routing bgp peer add name=peer1 remote-address=192.168.1.2 remote-as=65002
# Menambahkan peer BGP dengan remote AS 65002
/routing bgp network add network=10.0.0.0/24
# Mengiklankan jaringan 10.0.0.0/24 ke peer BGP

Kesimpulan: Dari 2 pendapat tersebut dapat disimpulkan bahwa Dynamic Routing pada MikroTik merupakan metode di mana router secara otomatis mempelajari dan memperbarui informasi routing melalui protokol seperti RIP, OSPF, dan BGP. OSPF ideal untuk jaringan enterprise dengan kemampuan membagi jaringan menjadi area, sementara BGP digunakan untuk routing antar autonomous system seperti koneksi antar ISP. MikroTik RouterOS menyediakan implementasi lengkap dari protokol-protokol ini dengan perintah CLI yang komprehensif untuk konfigurasi dan pemantauan.

4. REDISTRIBUTION DAN ECMP

Redistribution Rute

Dari sumber Tech Layer-X (2025), redistribution adalah proses berbagi informasi routing antara protokol routing yang berbeda. MikroTik memungkinkan redistribusi rute connected, static, atau bahkan rute yang dipelajari melalui satu protokol dinamis ke protokol lain. Ini berguna dalam skenario di mana beberapa bagian jaringan menggunakan protokol routing yang berbeda.

/routing ospf instance set default redistribute-static=yes
# Meredistribusi rute statis ke OSPF
/routing ospf instance set default redistribute-connected=as-type-1
# Meredistribusi rute connected sebagai LSA tipe 1
/routing bgp instance set default redistribute-ospf=yes
# Meredistribusi rute OSPF ke BGP

Dari sumber MikroTik Forum (2025), redistribusi rute default ke OSPF dapat dilakukan dengan:

/routing ospf instance set default distribute-default=if-installed-as-type1
# Mendistribusikan rute default via OSPF sebagai External Type 1
# Opsi 'if-installed' berarti hanya didistribusikan jika rute default sudah ada

Gunakan redistribusi dengan hati-hati! Redistribusi yang tidak tepat dapat menyebabkan loop routing atau rute suboptimal. Selalu pertimbangkan untuk menerapkan filter rute dan manipulasi metrik untuk mengendalikan rute mana yang didistribusikan dan bagaimana mereka diprioritaskan.

ECMP (Equal Cost Multi-Path)

Dari sumber MikroTik Documentation (2024), ECMP adalah fitur yang memungkinkan router menggunakan lebih dari satu jalur dengan biaya (cost) yang sama ke tujuan yang sama. Ini memungkinkan load balancing traffic ke beberapa gateway. Rute ECMP memiliki beberapa nilai gateway next-hop, dan semua next-hop yang dapat dijangkau disalin ke FIB dan digunakan untuk meneruskan paket.

/ip route add dst-address=192.168.100.0/24 gateway=10.0.0.1,10.0.0.2
# Menambahkan rute ECMP dengan dua gateway
/ip route print
# Melihat tabel routing
Flags: ... + - ECMP
# DST-ADDRESS GATEWAY Distance
0 AS+ 192.168.100.0/24 10.0.0.1 1
10.0.0.2 1

Dari sumber MikroTik Wiki (2024), OSPF dan BGP dapat secara otomatis menciptakan rute ECMP jika mereka menemukan beberapa jalur dengan biaya yang sama ke tujuan yang sama. Rute-rute ini diberi flag "+" dalam output /ip route print dan dikelompokkan bersama secara otomatis oleh RouterOS.

Metode load balancing default untuk ECMP pada MikroTik adalah per-connection, yang berarti semua paket untuk koneksi yang sama akan mengikuti jalur yang sama. Ini memastikan bahwa paket tidak tiba di luar urutan yang dapat menyebabkan masalah untuk protokol seperti TCP.

/ip route rule add dst-address=192.168.100.0/24 action=lookup table=main routing-mark=ecmp-mark
# Menambahkan aturan routing untuk ECMP
/ip firewall mangle add chain=prerouting dst-address=192.168.100.0/24 action=mark-routing new-routing-mark=ecmp-mark passthrough=yes per-connection-classifier=both-addresses:2/0
# Konfigurasi load balancing per-connection dengan klasifikasi berdasarkan alamat sumber dan tujuan

Kesimpulan: Dari 2 pendapat tersebut dapat disimpulkan bahwa MikroTik mendukung redistribusi rute antar protokol routing, memungkinkan integrasi berbagai bagian jaringan yang menggunakan protokol berbeda. ECMP memungkinkan load balancing traffic melalui beberapa jalur dengan biaya yang sama, meningkatkan throughput dan redundansi. Fitur-fitur ini dapat dikonfigurasi melalui CLI MikroTik dengan berbagai opsi untuk mengontrol bagaimana rute dibagikan dan traffic didistribusikan, menjadikan MikroTik solusi routing yang fleksibel untuk berbagai skenario jaringan.

5. TROUBLESHOOTING ROUTING

Tools Diagnostik

Dari sumber MikroTik Documentation (2024), RouterOS menyediakan berbagai tools untuk mendiagnosis masalah routing. Tools ini membantu administrator mengidentifikasi dan mengatasi masalah yang berkaitan dengan konfigurasi routing.

/ping 8.8.8.8
# Memeriksa konektivitas dasar ke alamat IP tertentu
/tool traceroute 8.8.8.8
# Melacak rute yang diambil paket ke tujuan
/ip route print
# Melihat tabel routing lengkap
/ip route print detail where dst-address=192.168.1.0/24
# Melihat detail rute spesifik
/tool torch interface=ether1
# Memantau traffic secara real-time pada interface

Dari sumber Tech Layer-X (2025), untuk troubleshooting OSPF, perintah-perintah berikut sangat berguna:

/routing ospf neighbor print
# Memeriksa status neighbor OSPF
/routing ospf interface print
# Melihat interface OSPF
/routing ospf route print
# Melihat rute OSPF
/system logging add topics=ospf,debug
# Mengaktifkan debug logging untuk OSPF

Masalah Umum dan Solusinya

Dari sumber MikroTik Documentation (2024), beberapa masalah umum dalam routing termasuk:

  1. Gateway Tidak Dapat Dijangkau - Periksa konektivitas fisik dan konfigurasi IP
  2. Rute Tidak Aktif - Periksa flag 'I' (inactive) dan alasannya di detail rute
  3. Konflik Rute - Rute dengan prefix yang sama tetapi distance berbeda
  4. Masalah Redistribusi - Filter rute yang tidak benar atau metrik tidak sesuai

Dari sumber Tech Layer-X (2025), untuk mengatasi masalah OSPF:

  1. Neighbor Tidak Terbentuk - Periksa area ID, autentikasi, dan parameter Hello/Dead
  2. Rute Tidak Terpelajari - Periksa network statement dan filter rute
  3. Rute Tidak Menjadi Aktif - Periksa konflik dengan rute statis (distance)
  4. Loop Routing - Periksa konfigurasi redistribusi dan filter rute

Untuk mengatasi masalah gateway yang tidak dapat dijangkau tetapi diperlukan untuk default route, gunakan fitur check-gateway:

/ip route add dst-address=0.0.0.0/0 gateway=192.168.1.1 check-gateway=ping
# Menambahkan default route dengan pemantauan gateway

Best Practices

Dari sumber Tech Layer-X (2025), beberapa best practices untuk routing di MikroTik:

  • Selalu gunakan router-id statis untuk protokol routing dinamis
  • Implementasikan keamanan (autentikasi) pada protokol routing
  • Gunakan filter rute untuk kontrol atas rute yang diiklankan dan diterima
  • Pertahankan dokumentasi yang baik untuk topologi jaringan dan konfigurasi routing
  • Gunakan fitur monitoring untuk pemantauan proaktif

Dari sumber MikroTik Forum (2025), beberapa tips untuk routing yang efisien:

  • Pilih protokol routing yang sesuai dengan ukuran dan kompleksitas jaringan
  • Gunakan route summarization untuk mengurangi ukuran tabel routing
  • Implementasikan multiple default routes dengan prioritas berbeda untuk redundansi
  • Manfaatkan ECMP untuk load balancing dan high availability
  • Gunakan routing mark untuk implementasi policy routing yang lebih kompleks

Awasi resource router Anda! OSPF dan BGP dapat menggunakan CPU dan memori yang signifikan pada perangkat MikroTik, terutama pada jaringan besar. Gunakan perintah /routing stats memory untuk memantau penggunaan memori oleh protokol routing.

Kesimpulan: Dari 2 pendapat tersebut dapat disimpulkan bahwa troubleshooting routing pada MikroTik melibatkan penggunaan berbagai tools diagnostik seperti ping, traceroute, dan torch, serta perintah khusus untuk protokol routing seperti OSPF dan BGP. Masalah umum meliputi gateway tidak terjangkau, rute tidak aktif, konflik rute, dan masalah redistribusi. Mengikuti best practices seperti menggunakan router-id statis, implementasi keamanan, filter rute yang tepat, dan monitoring proaktif dapat membantu mencegah masalah dan memfasilitasi troubleshooting yang efisien.
